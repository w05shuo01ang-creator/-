# Security Policy

## 报告漏洞

请不要在公开 Issue 中提交 OpenID、云环境 ID、访问令牌、用户图片或其他生产数据。

维护者应在 GitHub 仓库的 Security 页面开启 Private vulnerability reporting，并通过私密渠道处理漏洞报告。

## 安全边界

- 客户端代码不可信，所有所有权检查都在 `memeApi` 云函数完成。
- 云数据库集合必须禁止客户端直接读写。
- 文本和图片均明确通过微信内容安全检测才会自动公开，异常转人工复核。
- GitHub 仓库不应包含真实 AppID 对应的私有配置、生产数据或管理员身份列表。

## 已知限制

- 云函数已校验 5 MB、JPG/PNG 魔数、图片尺寸、像素量、每日配额和重复内容，并自动移除隐私元数据；仍需配合云存储权限与费用配额。
- MVP 每次最多扫描最近 100 条公开内容，适合冷启动，不适合大规模数据。
- 自动审核异常和社区举报内容需要运营人员人工复核；正式运营仍应补充专用审核后台。
- 删除云文件失败时会记录错误日志，但数据库记录已经删除；应定期检查无引用文件。
- `wx-server-sdk 4.0.2` 的官方依赖树仍被 npm 标记为包含旧 Axios/Lodash 安全公告。项目不使用受影响的实时 WebSocket 模块，也不把用户 URL 传入 SDK；应持续跟踪微信 SDK 更新，不要执行会降级 SDK 的 `npm audit fix --force`。
