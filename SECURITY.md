# Security Policy

## 报告漏洞

请不要在公开 Issue 中提交 OpenID、云环境 ID、访问令牌、用户图片或其他生产数据。

维护者应在 GitHub 仓库的 Security 页面开启 Private vulnerability reporting，并通过私密渠道处理漏洞报告。

## 安全边界

- 客户端代码不可信，所有所有权检查都在 `memeApi` 云函数完成。
- 云数据库集合必须禁止客户端直接读写。
- 表情申请公开后默认进入人工审核，不会自动公开。
- GitHub 仓库不应包含真实 AppID 对应的私有配置、生产数据或管理员身份列表。

## 已知限制

- 客户端 5 MB 上传限制仍需配合云存储规则和资源配额。
- MVP 每次最多扫描最近 100 条公开内容，适合冷启动，不适合大规模数据。
- 人工审核需要运营人员在云开发控制台执行，正式运营应补充审核后台、审计日志和内容安全服务。
- 删除云文件失败时会记录错误日志，但数据库记录已经删除；应定期检查无引用文件。
