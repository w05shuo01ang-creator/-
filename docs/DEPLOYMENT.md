# 部署与安全配置

## 1. 数据库集合

创建以下集合：

- `memes`：表情主体数据
- `likes`：用户对整张表情的点赞记录
- `tag_likes`：用户对标签的点赞记录

建议索引：

| 集合 | 字段 | 方向 |
| --- | --- | --- |
| memes | `isPublic`, `reviewStatus`, `createTime` | 前两项升序，时间降序 |
| memes | `_openid`, `createTime` | OpenID 升序，时间降序 |
| likes | `memeId`, `_openid` | 均升序 |
| tag_likes | `memeId`, `_openid` | 均升序 |

## 2. 数据库权限

小程序的所有数据访问都通过 `memeApi` 云函数完成。三个集合应设置为“所有用户不可读写”，不要开放客户端直读或直写。

云函数使用服务端身份访问数据库，不受客户端权限规则限制。

## 3. 云存储

上传路径格式为：

```text
uploads/{当前用户身份摘要}/{时间戳}_{随机串}.{扩展名}
```

上线前建议在云存储权限中只允许已登录用户上传，并限制单文件大小。客户端已限制为 5 MB，但客户端校验不能替代服务端或平台配额。

## 4. 内容审核

当前版本采用人工审核：

1. 用户上传后：`isPublic=false`，`reviewStatus=private`
2. 用户申请公开：`isPublic=false`，`reviewStatus=pending`
3. 管理员通过：`isPublic=true`，`reviewStatus=approved`
4. 管理员拒绝：`isPublic=false`，`reviewStatus=rejected`

正式运营时建议接入微信内容安全能力，并保留人工复核与投诉流程。自动审核异常时必须保持不公开。

## 5. 发布检查

- 配置合法 AppID 和生产云环境
- 部署云函数并安装云端依赖
- 验证三个集合对客户端完全关闭
- 验证上传、删除、申请公开、点赞、取消点赞
- 验证其他用户无法读取私密内容
- 配置云资源告警和每日费用上限
- 补充隐私保护指引、用户协议、内容规范和投诉入口
- 准备至少一组审核通过的初始化内容，避免首页首次进入为空
