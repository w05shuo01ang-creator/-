# 内容审核与用户上传安全

## 推荐审核状态机

```text
private
  -> pending
  -> auto_reviewing
  -> approved
  -> rejected
  -> manual_review
```

任何接口超时、回调验签失败、结果无法识别或系统异常，都保持非公开状态。不要在审核服务不可用时自动放行。

## 自动审核流程

1. 用户上传文件，记录保持 `isPublic=false`。
2. 服务端验证文件大小、真实文件类型、图片尺寸和归属路径。
3. 用户申请公开后，将记录更新为 `pending`。
4. 云函数提交微信内容安全的图片/媒体异步审核，并保存审核任务 ID。
5. 审核回调只写入审核结果和原因，不直接相信客户端传入结果。
6. 明确通过时设置 `reviewStatus=approved`、`isPublic=true`。
7. 明确违规时设置 `reviewStatus=rejected`、`isPublic=false`。
8. 疑似、接口异常或置信度不足时进入 `manual_review`。

正式接入时以微信官方当前版本的内容安全文档为准，优先使用异步媒体审核。审核调用、回调验签和状态修改都应放在云函数中。

## 用户上传风险

| 风险 | 影响 | 控制措施 |
| --- | --- | --- |
| 色情、暴力、违法或敏感内容 | 小程序处罚、下架或法律风险 | 自动审核、人工复核、失败关闭、举报与快速下架 |
| 版权、肖像权、隐私侵权 | 投诉、索赔 | 上传声明、投诉入口、证据留存、快速删除 |
| 伪造文件类型、超大图、畸形图片 | 解析异常、内存和费用消耗 | 服务端校验魔数、尺寸、像素量，安全解码并重新编码 |
| EXIF 定位和设备信息 | 用户隐私泄露 | 重新编码图片并移除 EXIF |
| 批量上传和刷接口 | 存储与云函数费用失控 | 按 OpenID 限流、每日额度、重复图片哈希、费用告警 |
| 私密图片越权访问 | 数据泄露 | 数据库禁止客户端直读，服务端鉴权，短期临时 URL |
| 点赞与标签刷量 | 排行失真 | 唯一点赞记录、事务计数、速率限制和异常检测 |
| 审核误判 | 正常内容被拒或违规内容漏过 | 申诉、人工复核、多级处置和审核日志 |

## MVP 上线前最低要求

- 每个 OpenID 每日上传数量和总容量限制
- 服务端验证真实图片类型、文件大小和最大像素量
- 自动审核异常时保持 `pending`
- 管理员审核页面和不可篡改的审核日志
- 用户举报、内容下架和账号封禁能力
- 隐私保护指引、用户协议和内容规范
- 云资源预算告警、日志告警和无引用文件清理任务
